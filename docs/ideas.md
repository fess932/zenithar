## Треды на сообщения (как в Mattermost)

**Суть:** у любого сообщения можно открыть тред — отдельную ветку ответов,
которая не засоряет основную ленту. В таймлайне у корневого сообщения видно
бейдж «N ответов» + аватары участников; клик открывает панель треда (сбоку на
десктопе, на весь экран на мобиле), где видны корень и все ответы по порядку.

**Почему это следующий логичный шаг:** фундамент уже есть — `messages.reply_to`
(см. ответы-цитаты, [REQUIREMENTS.md §3](../REQUIREMENTS.md)). Тред — это, по сути,
группировка сообщений по «корню ветки» поверх той же ссылки.

**Что нужно сделать (прикидка):**

- **Модель.** Ввести «корень треда». Варианты:
  - (a) переиспользовать `reply_to` и считать корнем самый верхний предок
    (рекурсивный CTE) — без новых колонок, но дороже на чтение;
  - (b) денормализовать `thread_root` (id корня) на сообщении — дешёвое чтение
    ветки одним `WHERE thread_root = ?`, но надо проставлять при отправке.
    **Скорее (b)** — наш паттерн (ср. денормализация `author_name`).
- **Счётчики.** `reply_count` + `last_reply_at` на корне (или вычислять). Нужны
  для бейджа и сортировки тредов по свежести.
- **Что показывать в основной ленте.** Решить: ответы в треде по умолчанию
  *не* попадают в главную ленту (mattermost-режим), либо есть тумблер «также
  отправить в канал». Это влияет на запрос истории (`recent_messages` фильтрует
  `thread_root IS NULL` для главной ленты).
- **WS-протокол.** Новый фрейм для подписки на тред (`thread-open`/`thread-history`)
  и доставки ответов открытого треда; либо расширить существующий fan-out полем
  `thread_root`, чтобы клиент сам решал, куда класть сообщение.
- **Фронт.** Панель треда (overlay/боковая колонка), бейдж «N ответов» на корне,
  переиспользовать `Composer` для ответа в тред. Mobile-first: тред на весь экран.
- **Звонки/вложения** в треде — наследуют существующую логику комнаты.

**Открытые вопросы:**

- Уведомления «вы участник треда» (требует presence/нотификаций — это Фаза 7).
- Глубина: плоский тред (как mattermost — все ответы к корню на одном уровне,
  без вложенности) сильно проще, чем дерево. **Начать с плоского.**
- Поиск/пагинация длинных тредов (пересекается с пагинацией истории).

**Объём:** средний. Бэкенд — одна колонка + пара запросов + фрейм; фронт — новая
панель + бейдж. Без вложенного дерева и нотификаций укладывается в одну фазу.
